# 自注意力教材

## 第一章：自注意力概念

### 1.1 定义

自注意力(Self-Attention)是一种注意力机制，允许序列中的每个位置关注序列中所有其他位置的信息。它是Transformer架构的核心组件，也是现代大型语言模型的基础。

### 1.2 设计动机

传统的序列处理模型（如RNN和CNN）存在以下问题：
- **RNN**：顺序计算，难以并行化，长距离依赖捕获能力有限
- **CNN**：感受野有限，难以捕获长距离关系

自注意力通过直接计算序列中任意两个位置之间的关系，解决了这些问题。

### 1.3 核心概念

- **Query (Q)**：当前位置的查询向量，表示"我要找什么"
- **Key (K)**：其他位置的键向量，表示"哪些位置有我需要的信息"
- **Value (V)**：其他位置的价值向量，表示"实际的信息内容"
- **注意力权重**：表示每个位置对当前位置的重要性

## 第二章：计算过程

### 2.1 基本公式

自注意力的计算公式为：

```
Attention(Q, K, V) = softmaxeft(rac{QK^T}{sqrt{d_k}}
ight)V
```

其中：
- Q、K、V 分别是查询、键、值矩阵
- d_k 是键向量的维度
- softmax 函数将注意力分数归一化为概率分布

### 2.2 详细步骤

1. **输入嵌入**：将输入序列转换为向量表示
2. **线性投影**：通过可学习的权重矩阵生成Q、K、V
   - Q = XW^Q
   - K = XW^K
   - V = XW^V
3. **计算注意力分数**：Q与K的点积
4. **缩放**：除以sqrt(d_k)，防止梯度消失
5. **应用掩码**：（可选）在解码器中使用，防止关注未来位置
6. **Softmax归一化**：将注意力分数转换为概率分布
7. **加权求和**：使用注意力权重对V进行加权求和

### 2.3 计算复杂度分析

- **时间复杂度**：O(n²d)，其中n是序列长度，d是嵌入维度
- **空间复杂度**：O(n²)，主要用于存储注意力权重矩阵

对于长序列，自注意力的计算复杂度可能成为瓶颈，因此出现了许多变体来优化这一点。

## 第三章：自注意力的优势

### 3.1 并行计算

自注意力可以并行处理整个序列，而不需要像RNN那样顺序计算，这大大提高了训练效率。

### 3.2 长距离依赖

自注意力直接计算序列中任意两个位置之间的关系，不受距离限制，可以有效捕获长距离依赖。

### 3.3 可解释性

注意力权重可以可视化，展示模型关注的位置，提高了模型的可解释性。

### 3.4 灵活性

自注意力适应不同长度的序列，不需要固定序列长度，这在处理变长序列时非常有用。

## 第四章：自注意力的局限性

### 4.1 计算复杂度

对于长序列，O(n²)的计算复杂度可能成为瓶颈，限制了模型处理长文本的能力。

### 4.2 位置信息缺失

自注意力本身不包含位置信息，需要额外的位置编码来提供序列的顺序信息。

### 4.3 内存消耗

存储注意力权重矩阵需要O(n²)的内存，对于长序列可能导致内存不足。

## 第五章：自注意力的变体

### 5.1 多头注意力

多头注意力将自注意力分成多个头，每个头关注不同的表示子空间，然后将结果拼接起来。这样可以捕获更丰富的特征。

### 5.2 线性注意力

线性注意力通过核函数将计算复杂度降低到O(n)，适用于长序列。

### 5.3 稀疏注意力

稀疏注意力只计算部分位置之间的注意力，减少计算量，例如：
- 局部注意力：只关注附近的位置
- 带状注意力：只关注特定范围内的位置
- 随机注意力：随机选择一些位置计算注意力

## 第七章：代码实现

### 7.1 基础实现

```python
def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    attention_scores = np.matmul(Q, K.T) / np.sqrt(d_k)
    
    if mask is not None:
        attention_scores = np.where(mask == 0, -np.inf, attention_scores)
    
    attention_weights = softmax(attention_scores)
    output = np.matmul(attention_weights, V)
    
    return output, attention_weights
```

### 7.2 实现技巧

- **数值稳定性**：使用softmax的数值稳定版本
- **矩阵运算**：使用高效的矩阵库（如NumPy、PyTorch）
- **内存优化**：对于长序列，考虑使用稀疏注意力
- **并行计算**：利用GPU加速矩阵运算

## 第八章：实验与分析

### 8.1 注意力可视化

通过热力图展示注意力权重，可以分析模型关注的位置，理解模型的决策过程。

### 8.2 超参数影响

- **嵌入维度**：更大的嵌入维度可以捕获更丰富的特征，但计算成本更高
- **序列长度**：长序列需要更多的计算资源，但可以捕获更多的上下文信息
- **注意力头数**：更多的注意力头可以捕获不同方面的信息，但会增加模型复杂度

## 第九章：总结与展望

### 9.1 总结

自注意力机制是现代深度学习的重要突破，它通过并行计算和直接捕获长距离依赖的能力，显著提升了模型性能。它已经成为Transformer架构的核心组件，被广泛应用于各种序列处理任务。

- **效率优化**：开发更高效的自注意力变体，以处理更长的序列
